# .goalie Directory - Production Cycle Telemetry & Governance

This directory contains telemetry, governance state, and ceremony outputs for the `af prod-cycle` workflow.

### CONSOLIDATED_ACTIONS.yaml Priority Levels

| Priority | Description | RCA Trigger |
|----------|-------------|-------------|
| `critical` | Immediate action required | 3+ consecutive failures, cross-circle cascade |
| `urgent` | Address within current iteration | Retry storms, safe-degrade overuse |
| `important` | Address in next planning cycle | Circle stagnation |
| `normal` | Standard backlog priority | Default for new items |
| `low` | Nice-to-have improvements | Manual assignment only |

### RCA Methods

| Method | Trigger Condition | Design Patterns |
|--------|-------------------|-----------------|
| `5-whys` | 3+ consecutive failures | circuit-breaker |
| `fishbone` | Retry storms (5+ attempts) | backoff-retry, jitter |
| `timeline-analysis` | Circle stagnation, cascading failures | circle-risk-focus, depth-ladder |
| `fault-tree` | Cross-circle cascade | bulkhead, governance-review |
| `value-stream-mapping` | Stagnation threshold reached | governance-review |

## Ceremony Outputs

Generated by `retro_coach.ts --ceremony-outputs`:

| File | Ceremony | Description |
|------|----------|-------------|
| `rca_findings.md` | RCA | Root cause analysis findings |
| `retro_summary.md` | Retrospective | What went well, improvements, action items |
| `standup_blockers.json` | Daily Standup | Critical/urgent blockers |
| `replenishment_queue.yaml` | Backlog Replenishment | Unverified high-priority actions |
| `refinement_candidates.yaml` | Backlog Refinement | Items needing refinement |
| `pi_sync_summary.md` | PI Planning | Cross-circle dependencies, velocity |
| `roam_risks.yaml` | Risk Management | ROAM-categorized risks |

## Governance Patterns (8 Total)

1. **depth-ladder** - Adaptive depth based on circle performance
2. **safe-degrade** - Graceful degradation on repeated failures
3. **circle-risk-focus** - Prioritize circles with highest ROAM delta
4. **autocommit-shadow** - Shadow mode for code autocommit
5. **guardrail-lock** - Lock guardrails on health degradation
6. **failure-strategy** - Abort/degrade strategies
7. **iteration-budget** - Dynamic iteration limits
8. **observability-first** - Telemetry before action

## Integration

### Enabling Ceremony Outputs

```bash
# Via CLI flag
af retro-coach --iteration 5 --run-id abc123 --ceremony-outputs

# Via environment variable
AF_CEREMONY_OUTPUTS=1 af prod-cycle 100
```

### Forensic Verification

Actions in CONSOLIDATED_ACTIONS.yaml are automatically verified using 24-hour pre/post windows:
- `verified: true` - Action showed measurable CoD/WSJF improvement
- `highImpact: true` - Frequency reduced by 20%+ post-action
