## 📋 Recommended Pre-Deletion Checklist

Before deleting ANY file:

- [ ] **Inspect Sample Content**: Use `head`, `jq`, `grep` to understand structure
- [ ] **Check Uniqueness**: Diff against current data
- [ ] **Extract Key Metrics**: Aggregate statistics before deletion
- [ ] **Assess Temporal Value**: Is there trend data?
- [ ] **Check References**: Are other files dependent on this?
- [ ] **Backup Strategy**: Archive or compress instead of delete
- [ ] **Document Decision**: Log why deletion is safe
- [ ] **Test Recovery**: Can you regenerate if needed?

## 🚨 RED FLAGS - Don't Delete If:

1. **Contains unique AI training data**
   - Pattern discoveries
   - Trajectory outcomes
   - Reinforcement learning feedback

2. **Has temporal patterns**
   - Trend analysis data
   - Baseline comparisons
   - Regression detection

3. **Required for compliance**
   - Audit trails
   - Incident records
   - Security logs

4. **Expensive to regenerate**
   - API calls with costs
   - Long computation times
   - External data sources

5. **Referenced by other systems**
   - Database foreign keys
   - Configuration files
   - Active cache references

## 💡 Better Alternatives to Deletion

1. **Compression**: Use gzip, bzip2, xz
   - Example: 145MB → 21MB (85% savings)

2. **Aggregation**: Summarize before deleting raw data
   - Daily summaries instead of per-second logs

3. **Archival**: Move to cold storage
   - AWS S3 Glacier
   - External drive
   - Cloud backup

4. **Rotation**: Keep last N days/files
   - Automatic via logrotate
   - Cron jobs for cleanup

5. **Refactoring**: Optimize storage format
   - JSON → MessagePack
   - Full content → References
   - Text → Binary format
